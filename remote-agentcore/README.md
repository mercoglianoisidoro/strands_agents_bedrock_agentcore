
# Remote AgentCore Package

Self-contained Bedrock AgentCore agent deployment package.

## Structure

```
remote-agentcore/
├── pyproject.toml              # Package configuration
├── pre-deploy.sh               # Workspace dependency sync script
├── remote_agents/
│   ├── remote_agent.py         # Main agent entrypoint
│   └── src/                    # Agent implementations
├── strands_shared/             # (generated) Copied from workspace for deployment
└── .bedrock_agentcore.yaml     # AgentCore configuration
```

## Setup

Install dependencies in isolated environment:

```bash
cd remote-agentcore
uv venv
source .venv/bin/activate
uv pip install -e .
```

## Local Development

Test locally before deploying:

```bash
agentcore dev
```

In another terminal, invoke:

```bash
agentcore invoke --dev '{"prompt": "hello"}'
```

## Deployment

### Pre-deploy: Sync Workspace Dependencies

This project depends on `strands-shared` from the monorepo workspace. Since agentcore's deployment packaging doesn't support uv workspace dependencies, you must sync them before deploying:

```bash
./pre-deploy.sh
```

This copies `strands_shared` from `../shared/` into this directory so it gets included in the deployment package.

**Run this script whenever:**
- You make changes to `strands-shared`
- Before each deployment

### Deploy to AWS

```bash
agentcore deploy
```

## Invoke Deployed Agent

```bash
agentcore invoke '{"prompt": "tell me a joke"}'
```

## Monitor Logs

```bash
aws logs tail /aws/bedrock-agentcore/runtimes/<agent-id>-DEFAULT --follow
```

## Stop Session

To stop active runtime sessions and save costs:

```bash
agentcore stop-session
```

## Notes

- `strands_shared/` is gitignored - it's generated by `pre-deploy.sh`
- For local development, the workspace dependency works via uv workspaces
- The pre-deploy step is only needed for cloud deployment
