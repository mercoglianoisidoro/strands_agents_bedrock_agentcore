
# Remote AgentCore

This package create and deploys Bedrock Agentcore agents, enabling serverless, scalable agent execution.
Unlike local agents that run on your machine, these agents are deployed to AWS and can be invoked via API.

## Prerequisites

Before deploying, ensure you have:

1. **AWS Account** with appropriate permissions:
   - Bedrock AgentCore access
   - Lambda function creation
   - IAM role creation
   - CloudWatch Logs access

2. **AWS CLI** configured:
   ```bash
   aws configure
   # Or verify existing configuration
   aws sts get-caller-identity
   ```

3. **Bedrock AgentCore CLI** installed:
   ```bash
   pip install bedrock-agentcore
   ```

4. **Workspace packages** installed:
   ```bash
   cd /path/to/strands_agents
   uv sync --all-packages
   ```

## Installation

This package is part of the strands_agents workspace and requires workspace dependencies.

### Workspace Dependency Sync

Since AgentCore's deployment packaging doesn't support UV workspace dependencies, you must sync `strands-shared` before deploying:

```bash
cd remote-agentcore
./pre-deploy.sh
```

**What this script does:**
1. Removes any existing `strands_shared/` directory
2. Copies fresh `strands_shared/` from `../shared/` into this directory
3. Creates `.bedrock_agentcore.yaml` configuration if missing

**When to run:**
- Before your first deployment
- After making changes to `strands-shared`
- Before each deployment (recommended)

**Note:** `strands_shared/` is gitignored as it's generated by this script.

## Local Development

Test your agent locally before deploying to AWS:

### Start Development Server

```bash
cd remote-agentcore/remote_agents
agentcore dev
```

This starts a local development server that simulates the AgentCore runtime.

### Test Invocation

In another terminal:

```bash
agentcore invoke --dev '{"prompt": "hello"}'
```

You should see the agent's response streamed back.

## Deployment

### Deploy to AWS

```bash
cd remote-agentcore/remote_agents
agentcore deploy
```

**What happens during deployment:**
1. Code is packaged into a zip file
2. Dependencies are resolved from `pyproject.toml`
3. Lambda function is created/updated
4. Bedrock AgentCore runtime is configured
5. `.bedrock_agentcore.yaml` is updated with agent ID

**First deployment** may take 2-3 minutes. Subsequent deployments are faster.

### Invoke Deployed Agent

```bash
agentcore invoke '{"prompt": "tell me a joke"}'
```

### Check Deployment Status

```bash
agentcore status
```

This shows:
- Agent ID
- Deployment status
- Lambda function details
- Last deployment time

## Monitoring & Management

### View Logs

**Using AgentCore CLI:**
```bash
agentcore obs logs
```

**Using AWS CLI:**
```bash
# Get agent ID from .bedrock_agentcore.yaml
aws logs tail /aws/bedrock-agentcore/runtimes/<agent-id>-DEFAULT --follow --region us-west-2
```

### Stop Active Sessions

To stop active runtime sessions and save costs:

```bash
agentcore stop-session
```

This terminates the Lambda execution environment. The agent will cold-start on the next invocation.

### Destroy Deployment

To completely remove the agent and all AWS resources:

```bash
agentcore destroy
```

**Warning:** This is irreversible and will delete:
- Lambda function
- IAM roles
- CloudWatch log groups
- AgentCore runtime configuration

## Configuration

### Environment Variables

The agent loads configuration from environment variables (set in Lambda):

```bash
# Deploy with custom environment variables
agentcore deploy --env MODEL_ID=us.anthropic.claude-3-5-sonnet-20241022-v1:0 \
                 --env AWS_REGION=us-west-2
```

**Available variables:**
- `MODEL_ID` or `CLAUDE_MODEL`: Bedrock model ID
- `AWS_REGION`: AWS region for Bedrock
- `SYSTEM_PROMPT`: Optional system prompt override
- `LOG_LEVEL`: Logging level (default: INFO)
